git常用命令

Posted on 2020-12-14 Edited on 2022-05-12

git命令

  • 新建分支

    git branch <分支名>

  • 新建分支并且切到当前分支

    git checkout -b <分支名>

gerrit相关命令

  • git push origin HEAD:refs/for/分支名

    当前commit推送到服务器后，需要code review

  • gerrit review提交重复

    首先，新增一个commit。
    多次执行 ‘git push origin HEAD:refs/for/分支名’ 命令后他人审核过代码后
    本地git pull命令，发现还会有上面的commit。
    可以强制把本地分支头重新置到远程分支头；命令：git reset –hard “commit的id”

git commit -am 与 -m 之前的区别

  • -am的意义 如果文件是已经新增到版本控制，后面文件修改则可省略git add 文件名，直接使用git commit -am “xxx”

git commit -m 与 git commit -am 的区别

起源

早上看到 东野圭吾的经典语录，有些句子很有感触，当然，在其他地方看到好的语录，我也会收藏起来，只是不去深思罢了。比如,抖音上，网易新闻上。。。其实现在信息大爆炸，可以学的地方很多，关键是找对或者找好一个点并且能够深入进去。想想以前听一个学长讲的，一句话：杀不死我的，都会使我更加强大，把它作为qq的签名，后面发现是其他地方看来的，好像是一部电影，这都不重要。到后来微信的：宁愿跑起来被绊倒无数次，也不要规规矩矩走一辈子。 仿佛还在昨天，到今天的 闭上眼睛，好好回想之前的努力，自信会喷涌而出。

如何去做

想要改变自己，首先需要从小的生活习惯开始，我发现如果一个人需要空出时间去做事，一定要去改变原先的生活方式，不然，会是三天打鱼，两天撒网的结果。

  • 明确自己要去做的事，并尽量避免或者不做浪费时间的事情

这样一来的好处是，你根本没有时间去做哪些没有营养的事情，脑子里短时间也不会想到这些东西。但是如果你没坚持住的话，马上又会变成老样子了。下面，说下更有用的方法。

  • 明确自己要去做的事后，在做事中寻找乐趣，就是找到一种成就感的东西，或许有点抽象，打个比方，就是你写代码写逻辑写了很久，到晚上，把应用或者成果展示出来的那种收获感。

这里的难点是往往乐趣或者成就感，不会马上获得，但是，一段时间，做了以后，就会有收获。对了，说下兴趣，其实如果如果一个东西迟迟没有结果，兴趣也会消失。以上是我个人愚见，仅供参考。
